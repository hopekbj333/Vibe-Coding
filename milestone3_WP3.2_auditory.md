# 📦 WP 3.2: 청각/순차 처리 훈련 (Auditory Sequential)

소리의 순서와 패턴을 기억하고 재현하는 훈련입니다.

---

## 💡 비개발자를 위한 전체 설명

**뭘 만드나요?**
소리를 순서대로 기억하고, 패턴을 인식하는 능력을 키우는 게임들입니다.

**왜 필요한가요?**
- 말소리를 순서대로 처리해야 단어/문장을 이해함
- "사과"를 듣고 → "사" 다음 "과"를 순서대로 처리
- 이 능력이 부족하면 듣기/받아쓰기가 어려움

**일상 예시:**
- 전화번호 불러주면 순서대로 기억
- 선생님 설명 순서대로 따라가기
- 동요 가사 순서 기억하기

---

## 🎵 시나리오 1: 소리 순서 기억

### S 3.2.1: 악기 순서 게임

**📋 기술적 요구사항:**
- 3~5개 악기 소리 순서대로 재생 → 순서대로 터치
- 게임화: 오케스트라 지휘자 컨셉
- 난이도: 악기 수 증가, 유사한 소리 추가

**💡 비개발자를 위한 설명:**

**어떻게 작동하나요?**
```
[소리] 🎹피아노 → 🥁드럼 → 🎸기타
           ↓
[화면] 여러 악기 그림
           ↓
[터치] 🎹 → 🥁 → 🎸 순서대로
           ↓
✅ 완벽한 연주!
```

**게임 컨셉:**
- "지휘자가 되어 악기들을 순서대로 연주시켜요!"
- 정답 시 오케스트라 합주 애니메이션
- 오답 시 어색한 소리 효과

**난이도:**
| 레벨 | 악기 수 | 특징 |
|------|---------|------|
| 1 | 3개 | 확연히 다른 소리 |
| 2 | 4개 | 약간 비슷한 소리 추가 |
| 3 | 5개 | 빠른 속도 |
| 4 | 5개 | 유사한 소리 (피아노 vs 실로폰) |

---

### S 3.2.2: 동물 소리 이야기

**📋 기술적 요구사항:**
- 동물 소리 시퀀스로 짧은 이야기 구성
- "먼저 강아지가 짖고, 그다음 고양이가..." → 순서 재현

**💡 비개발자를 위한 설명:**

**어떻게 작동하나요?**
```
[이야기]
"농장에서 아침이 밝았어요.
 먼저 🐓닭이 울고, 🐕강아지가 짖고, 🐄소가 울었어요"

[소리] 꼬끼오~ → 멍멍! → 음메~
           ↓
[터치] 🐓 → 🐕 → 🐄
           ↓
✅ 농장 아침 완성!
```

**스토리텔링:**
- 단순 나열이 아닌 이야기로 맥락 제공
- 아이가 더 쉽게 기억할 수 있음

---

### S 3.2.3: 사이먼 세즈 (Simon Says)

**📋 기술적 요구사항:**
- 화면 4분할, 순서대로 불빛 + 소리
- 같은 순서로 터치하여 재현
- 클래식 게임의 검증된 훈련 효과

**💡 비개발자를 위한 설명:**

**어떻게 작동하나요?**
```
[화면 4분할]
┌───┬───┐
│🔴│🔵│
├───┼───┤
│🟢│🟡│
└───┴───┘

[시연] 🔴(삐!) → 🟢(뿅!) → 🔵(띵!)
           ↓
[따라하기] 터치: 🔴 → 🟢 → 🔵
           ↓
✅ 레벨 업! 다음은 4개
```

**왜 효과적인가:**
- 1978년부터 사용된 검증된 기억력 게임
- 시각(불빛) + 청각(소리) 동시 자극
- 순서 기억 훈련에 최적

**난이도:**
- 레벨 1: 2개 순서
- 레벨 2: 3개 순서
- ...
- 최고 기록: N개 순서

## 🔄 시나리오 2: 패턴 인식

### S 3.2.4: 리듬 패턴 완성

**📋 기술적 요구사항:**
- 반복되는 리듬 패턴 중 빠진 부분 채우기
- 예: 쿵-짝-쿵-? → 짝

**💡 비개발자를 위한 설명:**

**어떻게 작동하나요?**
```
[소리] 쿵! 짝! 쿵! ???
           ↓
[화면] 다음에 올 소리는?
           ↓
[선택] [쿵!] [짝!]
           ↓
✅ 짝! 리듬 완성!
```

**패턴 예시:**
- 쉬움: 쿵-짝-쿵-짝 (ABAB)
- 보통: 쿵-쿵-짝-쿵-쿵-짝 (AABAA?)
- 어려움: 쿵-짝-짝-쿵-짝-짝 (ABB)

---

### S 3.2.5: 소리 규칙 찾기

**📋 기술적 요구사항:**
- 소리 시퀀스의 규칙 파악 후 다음 소리 예측
- 예: 높음-낮음-높음-? → 낮음

**💡 비개발자를 위한 설명:**

**어떻게 작동하나요?**
```
[소리] ↑높음 → ↓낮음 → ↑높음 → ???
           ↓
[화면] 다음 소리는?
           ↓
[선택] [↑높음] [↓낮음]
           ↓
✅ 낮음! 규칙 발견!
```

**패턴 유형:**
- 높낮이: 높-낮-높-낮
- 길이: 길-짧-길-짧
- 복합: 높고긴-낮고짧-높고긴-?

## 🎨 UI/UX 고려사항

**아동 친화적 디자인:**
- 밝고 귀여운 악기/동물 이미지
- 터치 시 즉각적인 피드백 (소리 + 애니메이션)
- 실패해도 격려 메시지

**접근성:**
- 소리 크기 조절 가능
- 재생 속도 조절 (느리게 듣기 옵션)
- 시각적 표시와 소리 동시 제공
