# 📖 스토리형 검사 설계 (Story-Based Assessment)

**작성일:** 2025-12-08  
**목적:** 표준화보다 학습에 집중한 스토리형 검사 시스템 설계  
**철학:** 검사를 "시험"이 아닌 "여행"처럼, 자연스러운 맥락에서 능력 측정

---

## 🎯 핵심 방향성

### 표준화 vs 학습 중심

**기존 접근 (표준화 중심)**:
- ❌ 고정된 문항 순서
- ❌ 표준 점수/규준 산출
- ❌ 타당도/신뢰도 검증에 집중
- ❌ "시험" 같은 분위기

**새로운 접근 (학습 중심)**:
- ✅ 스토리/여행 맥락에서 자연스러운 검사
- ✅ 아동의 현재 수준 파악에 집중
- ✅ 검사 결과를 학습 경로 설계에 활용
- ✅ "놀이" 같은 분위기

---

## 📚 스토리 구조 설계

### 기본 개념

검사를 **모험 여행**으로 구성:
- 아동이 주인공이 되어 여행을 떠남
- 각 검사 문항이 여행 중 만나는 "도전 과제"로 제시
- 정답/오답이 여행의 성공/실패로 표현
- 최종 결과가 "여행 완료" 및 "다음 여행 안내"로 제시

### 스토리 예시: "한글 나라 모험"

```
🎬 인트로
"안녕! 나는 한글 나라의 친구야. 
오늘 우리 함께 신비한 섬을 탐험해볼까?
섬에는 재미있는 퀴즈들이 있어.
하나씩 풀어보면서 우리의 실력을 알아볼게!"

🌴 Chapter 1: 소리 섬 (음운인식)
"첫 번째 섬은 '소리 섬'이야.
여기서는 소리를 듣고 구별하는 실력을 테스트해볼게."

🌊 Chapter 2: 기억 바다 (음운처리)
"다음은 '기억 바다'야.
소리를 기억하고 빠르게 말하는 실력을 확인해볼게."

🏔️ Chapter 3: 글자 산 (문자지식)
"세 번째는 '글자 산'이야.
글자를 보고 소리를 맞추는 실력을 알아볼게."

🌲 Chapter 4: 숲 속 퀴즈 (인지기초)
"마지막은 '숲 속 퀴즈'야.
집중력과 기억력을 테스트해볼게."

🎉 엔딩
"와! 모든 섬을 다 돌았어!
너의 실력은 [결과]야.
다음에는 [추천 학습]을 해보면 좋을 것 같아!"
```

## 🎮 검사 진행 방식

### 1. 스토리 기반 문항 제시

**기존 방식**:
```
Q1. 다음 중 같은 소리를 찾으세요.
[객관식 선택]
```

**스토리 방식**:
```
🎭 장면: 소리 섬의 동굴 안
👤 캐릭터: "이 동굴에는 마법의 소리들이 숨어있어.
           같은 소리를 가진 친구들을 찾아줘!"
🎵 소리 재생: "가방", "가지", "나무", "가로"
[객관식 선택 - 하지만 스토리 맥락에서]
```

### 2. 피드백 방식

**기존 방식**:
```
정답: ✓
오답: ✗
```

**스토리 방식**:
```
정답:
🎉 "훌륭해! 마법의 소리를 찾았어!"
✨ [시각적 효과: 별똥별, 반짝임]

오답:
💪 "아쉽지만 괜찮아! 다시 한 번 들어볼까?"
🔄 [재시도 기회 제공 - 학습 모드]
```

### 3. 진행 상황 표시

**기존 방식**:
```
진행률: 15/50 (30%)
```

**스토리 방식**:
```
🗺️ 지도 표시: 현재 위치 [소리 섬] → 다음 [기억 바다]
📊 진행률: 3/10 섬 완료
🎁 보상: 별 3개 획득!
```

## 🏗️ 시스템 구조

### 데이터 모델

```dart
// 스토리 기반 검사 세션
class StoryAssessmentSession {
  final String sessionId;
  final String childId;
  final StoryTheme theme; // "한글 나라", "우주 탐험" 등
  final List<StoryChapter> chapters; // 각 분야별 챕터
  final StoryProgress progress; // 현재 진행 상황
  final DateTime startedAt;
  final AssessmentMode mode; // tutorial, test
}

// 스토리 챕터
class StoryChapter {
  final String chapterId;
  final String title; // "소리 섬"
  final String description;
  final List<StoryQuestion> questions; // 해당 분야 문항들
  final StoryReward reward; // 완료 시 보상
}

// 스토리 문항
class StoryQuestion {
  final String questionId;
  final String storyContext; // "동굴 안에서..."
  final String characterDialogue; // 캐릭터 대사
  final QuestionModel question; // 실제 검사 문항
  final StoryFeedback feedback; // 피드백 메시지
}
```

## 🎨 UX 설계 원칙

### 1. Zero-Text Interface (아동 모드)
- 모든 지시는 **음성(TTS)**으로 제공
- 캐릭터가 직접 말하는 것처럼 표현
- 텍스트는 최소화 (부모/교사 모드에서만 상세 정보)

### 2. Cognitive Ease
- 화면당 **하나의 정보**만 제시
- 애니메이션 **1.5배 느리게**
- 스토리 전환 시 **페이드 효과** 사용

### 3. Gamification
- 각 챕터 완료 시 **보상** (별, 배지 등)
- 진행 상황을 **지도**로 시각화
- 실패해도 **격려 메시지** 제공

### 4. 학습 중심 피드백
- 정답: "왜 정답인지" 간단히 설명
- 오답: "다시 생각해볼까?" + 힌트 제공
- 재시도 기회 제공 (학습 모드)

## 📊 검사 결과 활용

### 표준화 지표 대신 학습 지표

**기존 방식**:
```
표준 점수: 85점
백분위: 60%
등급: 보통
```

**학습 중심 방식**:
```
🎯 현재 수준: 
   - 음운인식: 초급 (음절 수준)
   - 음운처리: 중급 (단어 반복 가능)
   - 문자지식: 초급 (자음 인식 중)

📚 추천 학습:
   - 음절 합성 게임 (10개)
   - 비단어 반복 게임 (5개)
   - 자음-소리 대응 게임 (8개)

🗺️ 다음 여행:
   "기억 바다 깊은 곳" (음운처리 심화)
```

### 적응형 학습 경로 제안

검사 결과를 바탕으로:
1. **약점 영역** 파악
2. **강점 영역** 확인
3. **개인화된 학습 경로** 제시
4. **다음 검사 시점** 안내

## 🚀 구현 단계

### Phase 1: 기본 스토리 구조 (MVP)
- [ ] 스토리 테마 1개 구현 ("한글 나라 모험")

- [ ] 기존 검사 문항을 스토리 맥락으로 래핑

### Phase 2: 스토리 요소 강화
- [ ] 캐릭터 애니메이션
- [ ] 배경음악/효과음
- [ ] 보상 시스템 (별, 배지)
- [ ] 스토리 피드백 메시지 다양화

### Phase 3: 다중 스토리 테마
- [ ] 테마 2: "우주 탐험"
- [ ] 테마 3: "동물 왕국"
- [ ] 사용자가 테마 선택 가능

### Phase 4: 학습 연계 강화
- [ ] 검사 결과 → 학습 경로 자동 생성
- [ ] 약점 영역 집중 훈련 제안
- [ ] 학습 후 재검사 제안

## 💡 주요 차별점

### 기존 검사 시스템과의 차이

| 항목 | 기존 검사 | 스토리형 검사 |
|------|----------|---------------|
| **목적** | 표준화된 평가 | 학습 수준 파악 |
| **분위기** | 시험/평가 | 놀이/모험 |
| **피드백** | 정오답만 | 학습적 피드백 |
| **결과 활용** | 점수/등급 | 학습 경로 제안 |
| **재시도** | 불가능 | 가능 (학습 모드) |
| **맥락** | 문맥 없음 | 스토리 맥락 |
