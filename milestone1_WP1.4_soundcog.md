# 📦 WP 1.4: 검사 모듈 A - 음운 인식 (Phonological Awareness)

PRD에서 가장 핵심이자 단계가 가장 많은 영역입니다. 5단계 구조에 맞춰 구성합니다.

## 🔊 시나리오 1: 1단계 - 청각적 주의 및 운율 (Listening & Prosody)

- [x] **S 1.4.1**: 소리 식별 ✅
    - 두 가지 소리(예: 북소리, 피아노 소리)를 순차 재생 후 "같아? 달라?" 질문 → 이미지 버튼(👍/👎) 터치

**💡 비개발자를 위한 설명:**

**뭘 했나요?**
아이에게 두 가지 소리를 들려주고, "같은 소리야? 다른 소리야?"를 맞추는 문제 유형을 만들었습니다.

**구체적으로:**
1. **소리 재생**: "첫 번째 소리" → (북소리) → "두 번째 소리" → (피아노 소리) 순서로 재생
2. **시각적 피드백**: 현재 재생 중인 소리 카드가 커지고 테두리가 강조됨
3. **다시 듣기**: 소리를 다시 듣고 싶으면 "다시 듣기" 버튼 클릭
4. **응답 버튼**: 👍 같아요 / 👎 달라요 큰 버튼으로 선택

**왜 필요한가요?**
- **청각적 주의력 측정**: 아이가 소리를 집중해서 들을 수 있는지 확인
- **음운 인식의 기초**: 다른 소리를 구분하는 능력은 글자 소리를 배우는 첫 단계
- **문해력 발달 예측**: 소리 구분 능력이 좋은 아이가 읽기 학습도 잘한다는 연구 결과

**비유로 설명하면:**
음악 시간에 선생님이 "첫 번째 악기 소리와 두 번째 악기 소리가 같니?"라고 물어보는 것과 같습니다.

---

- [x] **S 1.4.2**: 리듬 따라하기 ✅
    - 짧은 리듬 패턴(예: 똑-똑-똑) 재생 후, 아동이 화면을 탭하여 리듬 재현 → 탭 타이밍 기록

**💡 비개발자를 위한 설명:**

**뭘 했나요?**
화면에 리듬 패턴(똑-똑-똑)을 보여주고, 아이가 똑같은 리듬으로 화면을 터치하는 문제 유형을 만들었습니다.

**구체적으로:**
1. **리듬 시연**: 화면에 동그라미가 깜빡이면서 리듬을 보여줌 (예: ⚫ - ⚫ - ⚫)
2. **시각적 피드백**: 시연 중인 동그라미가 파란색으로 커지고 빛남
3. **따라하기**: 시연이 끝나면 큰 파란 원이 나타나고 "여기를 눌러봐!" 표시
4. **진행 표시**: 몇 번째 탭인지 동그라미에 체크 표시
5. **타이밍 기록**: 각 탭의 정확한 시간(밀리초)을 기록하여 리듬 정확도 평가

**왜 필요한가요?**
- **청각적 기억력 측정**: 리듬 패턴을 기억하고 재현하는 능력 확인
- **시간 감각 발달**: 일정한 간격을 인식하고 재현하는 능력 (읽기 리듬과 연관)
- **운율 인식의 기초**: 말의 리듬과 박자를 이해하는 능력 (문장 읽기의 기초)
- **집중력 평가**: 패턴을 끝까지 기억하고 수행하는 집중력

**비유로 설명하면:**
음악 시간에 선생님이 손뼉을 "짝-짝-짝" 치면, 아이도 똑같이 "짝-짝-짝" 따라 치는 것과 같습니다. 빠르기와 간격이 비슷한지 확인합니다.

---

- [x] **S 1.4.3**: 억양/강세 식별 ✅
    - 같은 문장을 다른 억양(평서/의문)으로 재생 후 "어떤 느낌이야?" → 표정 아이콘(😐/❓) 선택

**💡 비개발자를 위한 설명:**

**뭘 했나요?**
화면에 문장(예: "밥 먹었어?")을 보여주고, 그 문장이 "그냥 말하는 건지" 아니면 "물어보는 건지" 느낌을 맞추는 문제 유형을 만들었습니다.

**구체적으로:**
1. **문장 표시**: 큰 글씨로 문장이 나타남 (예: "밥 먹었어?")
2. **억양 힌트**: 문장 아래에 이모지로 힌트 제공 (😐 평서문, ❓ 의문문)
3. **느낌 선택**: 두 가지 버튼 중 선택
   - 😐 "그냥 말하는 거" (평서문)
   - ❓ "물어보는 거" (의문문)
4. **페이드 인 효과**: 문장이 서서히 나타나서 집중도 향상

**왜 필요한가요?**
- **운율 인식 능력**: 같은 단어라도 억양에 따라 의미가 달라지는 것을 이해
- **문장 이해의 기초**: 문장의 종류(평서문/의문문)를 구분하는 능력
- **읽기 유창성**: 글을 읽을 때 적절한 억양으로 읽는 능력의 기초
- **의사소통 능력**: 말의 느낌을 파악하는 능력 (사회성 발달과도 연관)

**비유로 설명하면:**
같은 말 "밥 먹었어"도 "밥 먹었어." (알려주는 말)와 "밥 먹었어?" (물어보는 말)가 다릅니다. 아이가 이 차이를 느낄 수 있는지 확인합니다.

## 🧱 시나리오 2: 2단계 - 단어 및 문장 구조 (Structure)

- [x] **S 1.4.4**: 단어 경계 인식 ✅
    - 문장 오디오 재생 후 "몇 단어야?" → 숫자 아이콘(1~5) 터치

**💡 비개발자를 위한 설명:**

**뭘 했나요?**
화면에 문장(예: "나는 학교에 갔어")을 보여주고, 그 문장이 몇 개의 단어로 이루어졌는지 숫자(1~5)로 선택하는 문제 유형을 만들었습니다.

**구체적으로:**
1. **문장 표시**: 큰 글씨로 문장이 나타남 (헤드폰 아이콘과 함께)
2. **페이드 인 효과**: 문장이 서서히 나타나서 집중도 향상
3. **숫자 선택**: 1~5까지 큰 버튼으로 제공
4. **시각적 피드백**: 선택한 숫자 버튼이 파란색으로 강조됨

**왜 필요한가요?**
- **단어 단위 인식**: 문장이 단어들의 조합이라는 것을 이해하는 능력
- **읽기의 기초**: 글을 읽을 때 단어 단위로 끊어 읽는 능력
- **문장 구조 이해**: 문장 구조를 파악하는 기초 능력
- **음운 인식 발달**: 말의 단위를 구분하는 능력 (음절 → 단어 → 문장)

**비유로 설명하면:**
레고 블록으로 만든 탑을 보고 "몇 개의 블록으로 만들었어?"라고 묻는 것과 같습니다. 문장이 여러 단어(블록)로 만들어졌다는 것을 이해하는지 확인합니다.

**예시:**
- "나는 학교에 갔어" = 3개 단어 (나는 / 학교에 / 갔어)
- "사과" = 1개 단어
- "고양이가 물을 마셔" = 3개 단어 (고양이가 / 물을 / 마셔)

---

- [x] **S 1.4.5**: 각운/두운 찾기 ✅
    - 세 단어 오디오(예: 사과, 바나나, 수박) 재생 후 "같은 소리로 시작하는 거 찾아봐" → 이미지 선택

**💡 비개발자를 위한 설명:**

**뭘 했나요?**
먼저 한 단어(예: "사과")를 보여주고, 그 단어와 같은 소리로 시작하는 단어를 보기에서 찾는 문제 유형을 만들었습니다.

**구체적으로:**
1. **기준 단어 제시**: 질문에 기준 단어 표시 (예: "사과")
2. **보기 제공**: 3개 단어 그림 버튼 (예: 사탕, 바나나, 포도)
3. **첫소리 비교**: "사과"는 "ㅅ"으로 시작 → 보기에서 "사탕"도 "ㅅ"으로 시작
4. **정답 선택**: "사탕" 그림을 선택하면 정답

**왜 필요한가요?**
- **음운 인식의 핵심**: 단어의 첫소리를 구분하는 능력 (난독증 예측 지표)
- **한글 학습 준비**: 자음(ㄱ, ㄴ, ㄷ 등)을 배우기 전 첫소리 인식 능력
- **읽기 발달**: 첫소리가 같은 단어를 빠르게 인식하는 능력 (읽기 유창성)
- **철자 인식**: 같은 소리 → 같은 글자라는 개념 이해

**비유로 설명하면:**
끝말잇기의 반대 버전입니다. 끝말잇기는 "끝 소리"를 찾지만, 이것은 "첫 소리"가 같은 것을 찾습니다.
- "사과", "바나나", "사탕" → "사과"와 "사탕"이 "ㅅ"으로 시작해서 같은 편!

## ✂️ 시나리오 3: 3단계 - 음절 조작 (Syllable Manipulation)

- [x] **S 1.4.6**: 음절 분해/합성 ✅
    - "나-비"를 천천히 재생 후 "합치면 뭐야?" → 그림 선택 (나비/나무/비행기)

**💡 비개발자를 위한 설명:**

**뭘 했나요?**
단어를 음절로 나누어 보여주고(예: "나-비"), 그것을 합치면 무엇인지 그림에서 찾는 문제 유형을 만들었습니다.

**구체적으로:**
1. **음절 분리 표시**: "나", "비" 두 개의 박스로 분리해서 표시
2. **순차 강조**: 각 음절이 순서대로 커지면서 강조됨 (노란색)
   - "나" (1초) - "비" (1초)
3. **합성 질문**: "합치면 뭐야?"
4. **그림 선택**: 나비, 나무, 비행기 중 선택
5. **다시 보기**: 음절을 다시 볼 수 있는 버튼

**왜 필요한가요?**
- **음절 인식의 핵심**: 단어가 음절로 나뉜다는 것을 이해하는 능력
- **읽기의 기초**: 글자를 읽을 때 음절 단위로 읽는 능력
- **철자 학습 준비**: 음절 → 글자 대응 관계 이해의 기초
- **난독증 예측**: 음절 합성 능력이 읽기 발달과 강한 상관관계

**비유로 설명하면:**
퍼즐 조각을 맞추는 것과 같습니다. "나" 조각과 "비" 조각을 합치면 "나비"가 됩니다. 아이가 조각(음절)을 보고 전체(단어)를 떠올릴 수 있는지 확인합니다.

**예시:**
- "나-비" → 나비 ✓
- "바-나-나" → 바나나 ✓
- "학-교" → 학교 ✓

---

- [x] **S 1.4.7**: 음절 탈락 ✅
    - "사과에서 '과'를 빼면?" → 글자 선택 (사/과/사과)

**💡 비개발자를 위한 설명:**

**뭘 했나요?**
단어(예: "사과")에서 특정 음절(예: "과")을 빼면 무엇이 남는지 글자를 선택하는 문제 유형을 만들었습니다.

**구체적으로:**
1. **단어 표시**: 큰 글씨로 원래 단어 표시 (예: "사과")
2. **빼기 기호**: 빨간색 마이너스 아이콘 표시
3. **제거할 음절**: 빨간 박스로 제거할 음절 표시 (예: "과")
4. **질문**: "뭐가 남아?"
5. **글자 선택**: 사, 과, 사과 중 선택
6. **페이드 인**: 문제가 서서히 나타남

**왜 필요한가요?**
- **음절 조작 능력**: 단어의 일부를 제거하고 남은 것을 인식하는 능력
- **음운 인식의 심화**: 단어 구조를 이해하고 조작하는 고급 능력
- **읽기 유창성**: 단어를 유연하게 다루는 능력 (읽기 발달의 중요 지표)
- **난독증 예측**: 음절 탈락 능력이 읽기 장애 예측에 효과적

**비유로 설명하면:**
레고로 만든 "사과" 탑에서 "과" 블록을 빼면 "사" 블록만 남습니다. 아이가 머릿속에서 이런 조작을 할 수 있는지 확인합니다.

**예시:**
- "사과"에서 "과" 빼면 → "사" ✓
- "바나나"에서 "나" 빼면 → "바나" ✓
- "학교"에서 "교" 빼면 → "학" ✓

- [x] **S 1.4.8**: 음절 뒤집기 ✅
    - "나비를 거꾸로 하면?" → **음성 녹음** 입력

**💡 비개발자를 위한 설명:**

**뭘 했나요?**
단어(예: "나비")를 거꾸로 말하면 무엇인지 음성으로 녹음하는 문제 유형을 만들었습니다.

**구체적으로:**
1. **단어 표시**: 큰 글씨로 단어 표시 (예: "나비")
2. **뒤집기 아이콘**: 위아래 화살표로 '거꾸로' 표현
3. **질문**: "거꾸로 하면?"
4. **녹음 버튼**: 큰 파란 원형 마이크 버튼
5. **녹음 진행**: 버튼을 누르면 3초간 녹음 (빨간색으로 변하며 펄스 애니메이션)
6. **자동 완료**: 3초 후 자동으로 녹음 종료 및 다음 문제로 이동

**왜 필요한가요?**
- **음절 순서 인식**: 단어의 음절 순서를 이해하고 조작하는 능력
- **음운 기억**: 음절을 기억하고 순서를 바꾸는 작업 기억 능력
- **고급 음운 인식**: 읽기 능력이 높은 아이를 구별하는 심화 평가
- **뇌 처리 능력**: 음운 정보를 머릿속에서 조작하는 능력 (읽기와 강한 상관관계)

**비유로 설명하면:**
"나비"라는 기차에서 "나" 칸과 "비" 칸의 순서를 바꾸면 "비나"가 됩니다. 아이가 머릿속에서 이런 조작을 할 수 있는지 확인합니다.

**예시:**
- "나비" → 거꾸로 → "비나" ✓
- "고기" → 거꾸로 → "기고" ✓
- "토마토" → 거꾸로 → "토마토" ✓ (대칭)

**녹음 기능:**
- 실제로는 아이의 음성을 녹음하여 나중에 채점자가 듣고 평가
- 현재는 녹음 시뮬레이션 (3초 녹음 후 자동 완료)

## 🔬 시나리오 4: 4단계 - 음소 조작 (Phoneme Manipulation)

- [x] **S 1.4.9**: 초성 분리 ✅
    - "강아지의 첫 소리는?" → 음성 녹음 (ㄱ)

**💡 비개발자를 위한 설명:**

**뭘 했나요?**
단어(예: "강아지")의 첫 소리(초성)가 무엇인지 음성으로 녹음하는 문제 유형을 만들었습니다.

**구체적으로:**
1. **단어 표시**: 큰 글씨로 단어 표시 (예: "강아지")
2. **질문**: "첫 소리는?" (promptText에 표시)
3. **녹음 버튼**: 파란 마이크 버튼 클릭
4. **녹음 진행**: 3초간 빨간색으로 변하며 녹음
5. **정답 예시**: "ㄱ" 또는 "기역" 소리 녹음
6. **자동 완료**: 녹음 후 자동 제출

**왜 필요한가요?**
- **음소 인식의 시작**: 단어를 자음/모음 단위로 분해하는 능력
- **한글 학습 준비**: 자음(ㄱ, ㄴ, ㄷ)을 인식하고 발음하는 능력
- **난독증 핵심 지표**: 음소 인식 능력이 읽기 장애의 가장 강력한 예측 인자
- **철자 학습**: 소리 → 글자 대응 관계 이해의 기초

**비유로 설명하면:**
"강아지"라는 레고 탑에서 가장 아래 블록(첫 소리)만 떼어내는 것입니다. "ㄱ" 소리가 첫 블록입니다.

**예시:**
- "강아지" → 첫 소리 → "ㄱ" (기역) ✓
- "사과" → 첫 소리 → "ㅅ" (시옷) ✓
- "나비" → 첫 소리 → "ㄴ" (니은) ✓

**녹음 필요 이유:**
- 글자를 모르는 아이도 소리로 답할 수 있음
- 발음을 정확히 평가할 수 있음

- [x] **S 1.4.10**: 음소 합성/분할 ✅
    - "ㄱ-ㅏ 합치면?" → 글자 선택 (가위/나무/다리)

**💡 비개발자를 위한 설명:**

**뭘 했나요?**
자음과 모음을 따로 보여주고(예: "ㄱ-ㅏ"), 합치면 어떤 소리인지 단어를 선택하는 문제 유형을 만들었습니다.

**구체적으로:**
1. **음소 분리 표시**: "ㄱ", "ㅏ" 두 개의 박스로 분리
2. **순차 강조**: 각 음소가 순서대로 커지면서 강조됨
   - "ㄱ" (1초) - "ㅏ" (1초)
3. **합성 질문**: "합치면 뭐야?"
4. **글자 선택**: 가위, 나무, 다리 중 선택
5. **정답**: "가위" (ㄱ + ㅏ = 가로 시작)

**왜 필요한가요?**
- **음소 인식의 핵심**: 자음과 모음을 합쳐 글자를 만드는 능력
- **한글 읽기의 기초**: 한글은 자음+모음 조합으로 이루어짐을 이해
- **난독증 핵심 지표**: 음소 합성 능력이 읽기 성공의 가장 강력한 예측 인자
- **읽기 학습 준비도**: 이 능력이 있어야 한글을 배울 수 있음

**비유로 설명하면:**
"ㄱ" 블록과 "ㅏ" 블록을 합치면 "가" 글자가 만들어집니다. 마치 레고 두 조각을 합쳐서 새로운 모양을 만드는 것과 같습니다.

**예시:**
- "ㄱ" + "ㅏ" = "가" → 가위, 가방, 가족 등 "가"로 시작하는 단어
- "ㄴ" + "ㅏ" = "나" → 나무, 나비, 나라 등 "나"로 시작하는 단어
- "ㄷ" + "ㅏ" = "다" → 다리, 다람쥐 등 "다"로 시작하는 단어

**중요성:**
이 문제를 잘 못하는 아이는 한글 읽기 학습에 어려움을 겪을 가능성이 높습니다. 조기 발견이 중요합니다.

- [x] **S 1.4.11**: 음소 대치/추가 ✅
    - "공에서 'ㄱ'을 'ㅂ'으로 바꾸면?" → 음성 녹음

**💡 비개발자를 위한 설명:**

**뭘 했나요?**
단어에서 특정 소리를 다른 소리로 바꾸는 문제 유형을 만들었습니다. 예: "공"에서 "ㄱ"을 "ㅂ"으로 바꾸면 "봉"이 됩니다.

**구체적으로:**
1. **원래 단어 표시**: "공" (파란색 박스)
2. **빼기 표시**: 빨간색 빼기 아이콘 + "ㄱ" (빨간색 박스)
3. **더하기 표시**: 초록색 더하기 아이콘 + "ㅂ" (초록색 박스)
4. **마이크 버튼**: 누르면 3초간 녹음
5. **시각적 피드백**: 녹음 중 빨간색으로 깜빡임

**왜 필요한가요?**
- **음소 조작 능력**: 단어 내의 소리를 변형할 수 있는 고급 음운 인식 능력
- **읽기/쓰기 직결**: 철자법 이해와 단어 만들기에 필수
- **난독증 예측**: 이 능력이 부족하면 맞춤법과 읽기에 어려움 발생
- **메타인지 언어 능력**: 언어를 대상으로 생각할 수 있는 능력

**비유로 설명하면:**
레고 집에서 빨간 블록 하나를 빼고 파란 블록을 끼워 넣는 것과 같습니다. 전체 구조는 유지하되 한 부분만 바뀝니다.

**예시:**
- "공" - "ㄱ" + "ㅂ" = "봉"
- "달" - "ㄷ" + "ㅁ" = "말"
- "곰" - "ㄱ" + "ㅅ" = "솜"

**중요성:**
이 능력은 단순 암기가 아닌 음운 규칙의 이해를 보여줍니다. 읽기 학습의 매우 높은 수준입니다.

## 🧠 시나리오 5: 5단계 - 음운 기억 (Phonological Memory)

- [x] **S 1.4.12**: 비단어(Non-word) 따라말하기 ✅
    - 존재하지 않는 단어(예: "두파리") 재생 후 → 음성 녹음

**💡 비개발자를 위한 설명:**

**뭘 했나요?**
실제로 존재하지 않는 단어(비단어)를 들려주고 그대로 따라 말하게 하는 문제입니다. 예: "두파리", "가토미"

**구체적으로:**
1. **자동 재생**: 0.5초 후 비단어 자동 표시
2. **말풍선 애니메이션**: 스피커 아이콘 + 비단어가 탄성 있게 나타남
3. **2초 표시**: 비단어를 2초간 보여줌
4. **마이크 버튼**: 누르면 3초간 녹음
5. **피드백**: 녹음 중 빨간색 깜빡임

**왜 필요한가요?**
- **순수 음운 처리**: 의미에 의존하지 않고 소리만으로 기억
- **음운 단기 기억**: 새로운 단어를 배울 때 필수적인 능력
- **어휘 학습 예측**: 이 능력이 좋으면 새 단어를 빨리 배움
- **읽기 난이도와 직결**: 처음 보는 단어를 읽는 능력과 관련

**비유로 설명하면:**
처음 들어보는 외국어 단어를 그대로 따라 말하는 것과 같습니다. 뜻은 몰라도 소리를 정확히 재현해야 합니다.

**예시:**
- "두파리" → 의미는 없지만 "두-파-리" 소리 구조는 한국어 규칙 따름
- "가토미" → 존재하지 않는 단어지만 발음 가능한 조합
- "비롱개" → 마찬가지로 비단어지만 한국어 음운 규칙에 맞음

**중요성:**
실제 단어는 의미로 도움을 받을 수 있지만, 비단어는 순수하게 음운 처리 능력만 측정합니다. 읽기 장애 진단의 핵심 지표입니다.

- [x] **S 1.4.13**: 숫자/단어 폭 기억 ✅
    - 숫자 시퀀스(예: 3-7-2) 재생 후 → 음성 녹음으로 따라 말하기
    - 단계별로 시퀀스 길이 증가 (3개 → 4개 → 5개)

**💡 비개발자를 위한 설명:**

**뭘 했나요?**
여러 개의 숫자나 단어를 순서대로 들려주고, 그 순서를 기억해서 따라 말하게 하는 문제입니다.

**구체적으로:**
1. **자동 재생**: 0.5초 후 시퀀스 재생 시작
2. **순차 표시**: 각 숫자가 1초씩 커지면서 강조됨
   - 예: "3" (1초) → "7" (1초) → "2" (1초)
3. **시각적 상태**: 현재 재생 중(파란색), 이미 재생됨(회색), 대기 중(연한 회색)
4. **마이크 버튼**: 재생 완료 후 마이크로 녹음
5. **3초 녹음**: "3, 7, 2" 순서대로 말하기

**왜 필요한가요?**
- **음운 작업 기억**: 정보를 단기간 유지하고 처리하는 능력
- **읽기 이해의 기초**: 문장을 읽을 때 앞부분을 기억하며 뒷부분 이해
- **학습 능력 예측**: 작업 기억이 좋으면 학습이 빠름
- **주의력과 연결**: ADHD와 읽기 문제가 겹치는 경우 이 능력 낮음

**비유로 설명하면:**
전화번호를 듣고 바로 눌러야 하는 상황과 같습니다. 짧은 시간 동안 여러 개의 정보를 순서대로 기억해야 합니다.

**예시:**
- **3개 (쉬움)**: 3-7-2
- **4개 (보통)**: 5-2-8-1
- **5개 (어려움)**: 4-9-1-6-3

**단어 버전도 가능:**
- **3개**: 강아지-사과-나무
- **4개**: 의자-공-연필-책

**중요성:**
작업 기억 용량이 작으면 긴 문장을 읽고 이해하기 어렵습니다. 읽기 이해력의 핵심 인지 자원입니다.
